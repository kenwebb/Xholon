Xholon Showcase

Xholon is a tool for working on whatever I'm interested in, and I hope that other people will also find it useful. The focus is on hierarchical structure, building complex worlds, open-ended dynamic interactions, visualization, and letting people paste in new content while the application is running. This page lists some of the applications I've developed over the years, converted to run using HTML5, JavaScript, CSS, SVG, and canvas. All of these apps run entirely locally in the browser, with no server processing required (except to initially download the web content). The apps require a modern browser (Firefox, Chrome, Opera, Safari, IE9), and a fairly high-speed internet connection. These sample apps are primarily intended to test and demonstrate ideas, approaches, and features for Xholon, and to explore other domains that I'm interested in. They are all incomplete.

To get started, you might like to look at some apps where graphics are especially important.

Any Xholon app can use a d3 circle-packing GUI. Some recent apps use d3 for interactive animations, such as for children's stories.

Any Xholon app can be exported as a Twine 2 interactive story. There are some initial test examples of this new capability.

You can create a Xholon Avatar with a behavior, using Google Blockly.

I've started building environments that engage pre-school and elementary school children in computer coding.

Xholon can run Scratch apps.

I would like to collaborate with artists.

Xholon apps can be exported as Leaflet maps. Here's an early example.

You can use Meteor with Xholon.

I'm working on collections of independent apps that use WebRTC to interact with each other.

Xholon can animate movie and TV screenplays, including those written in the popular Fountain format.

I am learning about Category Theory.

I am also learning about Operads, a sub-discipline of category theory.

Each app offers three Graphical User Interface (GUI) options.

  1. A classic tree-based view of the hierarchy (clsc).
  2. A contour view of the hierarchy, using the circle-packing layout from the D3 graphics library (d3cp).
  3. A minimal view that just runs the application (none).
